# 環境状態台帳(state.md)

各章を書き終えた時点で「その章終了時点に存在するリソースと設定」を追記していく台帳です。次章の執筆・レビュー時は、まずこのファイルで前章までの到達状態を確認します。

- 各章のセクションには「作成したリソース / RBAC・ポリシー変更 / ネットワーク変更 / コスト影響」を表で記録します。
- ハンズオンの内容と食い違いが出ないよう、章のマージ前に必ず更新します(CLAUDE.md のルール)。

## 第1章 終了時点

第1章では、土台の「器の骨格」を確定させました。dev/prod の分離方式(サブスクリプション分離を主軸)・命名規則(CAF 準拠)・主リージョン(東日本)を決め、最初のリソースグループを作成しています。

### 決めた方針(以降の章の前提)

| 項目 | 決定内容 |
|---|---|
| dev/prod 分離方式 | サブスクリプション分離を主軸(個人学習はリソースグループ分離に読み替え。付録E) |
| 命名規則 | CAF 準拠 `{種別}-{ワークロード}-{環境}-{リージョン}-{連番}`、ワークロード名は `foundation` |
| 主リージョン | 東日本(`japaneast`) |
| 必須タグ(項目のみ決定) | `env` / `owner`(強制は第3章のポリシーで実施) |

### 作成したリソース

| リソース種別 | 名前 | スコープ | 備考 |
|---|---|---|---|
| 管理グループ | `mg-foundation` | テナント | dev/prod サブスクリプションを束ねる統制の器 |
| リソースグループ | `rg-foundation-dev` | サブスクリプション(dev) | 東日本。タグ `env=dev` / `owner` |
| リソースグループ | `rg-foundation-prod` | サブスクリプション(prod) | 東日本。タグ `env=prod` / `owner` |
