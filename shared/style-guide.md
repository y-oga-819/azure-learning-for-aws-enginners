# 文体・表記ガイド

本シリーズ共通の執筆スタイルを定めます。用語の対訳・統一表記は `shared/glossary.md`(および `prh.yml`)を参照してください。ここでは文体とレイアウトの約束事を扱います。

## 文体

- **です・ます調**に統一します(「である調」は使いません)。
- 読者への呼びかけは「〜してみましょう」「〜します」を基本とし、過度にくだけた表現は避けます。
- 1 文はできるだけ短く。目安として 120 文字を超えたら分割を検討します(textlint が警告します)。
- 主語と述語の対応、二重否定、同じ助詞の連続に注意します。

## 用語

- 用語は `shared/glossary.md` に従います。初出時のみ英語併記(例:「マネージド ID(Managed Identity)」)。
- AWS 対比は各章の必須枠です。対応関係は対訳表と整合させます。

## コードブロック

- コードブロックには**必ず言語指定**を付けます(` ```bash ` / ` ```bicep ` / ` ```json ` / ` ```kql ` など)。
- シェル操作は原則 Azure CLI(`az ...`)を主とし、必要に応じてポータル手順を併記します。
- 実行結果を載せる場合は、`handson-logs/` の検証済みログから引用します(創作しない)。

## 画面キャプチャ・図版

- 画像は `assets/figures/` に置き、ファイル名は **`chXX-figYY-説明.png`** 形式にします(例:`ch04-fig01-vnet-address-design.png`)。
- 本文からは相対パスで参照します。参照切れは CI(`npm run check:links`)で検出されます。
- 図には必ずキャプション(`figcaption` 相当の説明)を付けます。

## コラム・注意書きの記法

紙面での装飾は `themes/techbook/theme.css` が担当します。原稿では次の HTML ブロックを使います。

コラム(補足・脱線・実案件の知識):

```html
<div class="column">
<span class="column-title">コラム:オンプレ AD との同期</span>

本文の流れから外れる補足をここに書きます。

</div>
```

注意・警告(課金事故・取り返しのつかない操作など):

```html
<div class="note">
<span class="note-title">注意:削除は取り消せません</span>

読者が事故を起こしやすい箇所で警告します。

</div>
```

## 検証マーカー

- ハンズオン手順・コマンドで検証ログの裏付けがないものには `<!-- 要検証 -->` を付けます。
- 料金・SKU 名・上限値・プレビュー/GA 状況などの数値・製品情報には `<!-- 要検証: 公式Doc確認 -->` を付けます。
- 残存件数は CI(`npm run check:todo`)で集計されます。**発行前は 0 件**を必達とします。

## 章の構成(必須の 5 部構成)

すべての章は次の順で書きます(`manuscripts/chNN.md` の雛形に反映済み)。

1. このリソースは何か
2. AWS との対比
3. 土台への適用(Before / After)
4. 設計の勘所
5. ハンズオン
